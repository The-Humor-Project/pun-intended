## Repository instructions

- Fonts: use only web-safe fonts; do not load fonts from external services or local font files.
- Allowed stacks: `Georgia, "Times New Roman", Times, serif`, `"Trebuchet MS", Verdana, Tahoma, sans-serif`, `"Courier New", Courier, monospace`, `"Lucida Sans", "Lucida Sans Unicode", "Lucida Grande", sans-serif`.

## Supabase schema
- The latest database types live in src/types/supabase.ts
- This is generated by running npm run supabase:types
- The user does not want to use a SUPABASE_ROLE_KEY.  All actions should use the logged-in user's JWT key.

## Markdown format
- Assignments, meeting agendas, and documentation content use Markdown-It default (GFM-like) with HTML, linkify, typographer, and hard line breaks enabled.
- Supported syntax includes headings, lists, fenced code blocks with language tags, inline code, links, blockquotes, tables, horizontal rules, images, and strikethrough.

## Design
- The non-admin area of the site should be very round and curvy
- The admin area of the site should be very sharp, with no round corners
- The non-admin area of the site should use the Columbia color palette.
- Columbia Blue: #B9D9EB
- Primary Blue: #1D4F91
- Secondary Blue 1 #003087
- Secondary Blue 2 #0077C8
- Secondary Blue 3 #6CACE4.
- Access Color 1: #C75000
- Neutral 1: #222222
- Neutral 2: #53565A
- Neutral 3: #D0D0CE
- Neutral 4: #F0F0F0
